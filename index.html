Simmit Pay 금융 플랫폼 기반 통합 인프라 및 보안 네트워크 구축

2026.04

가상의 금융회사 Simmit Pay를 주제로 Rocky Linux 기반의 서버와 기업 네트워크를 구축하고, 금융 서비스에 필요한 네트워크 분리, 접근 통제, 암호화 통신과 로그 관리 환경을 구현했습니다.

  • 팀장으로서 전체 프로젝트 흐름, 역할과 일정을 관리했습니다.
  • 부서별 VLAN, 라우팅, ACL, VPN과 NAT 정책을 설계하고 전체 구성을 검토했습니다.
  • 웹·DB·로그 서버 간 연결 상태를 확인하고 통합 과정의 오류를 분석·수정했습니다.
  • 네트워크 시연 영상, 회의록과 프로젝트 보고서 작성에 참여했습니다.

구축 내용

본사와 지사, 재택근무 환경을 가정하고 부서별 VLAN을 구성했습니다. VPN을 통한 암호화 통신, ACL 기반 최소 권한 접근 제어, NAT를 통한 내부 주소 은닉과 로그 수집 환경을 적용했습니다.

담당 역할

네트워크 설계와 전체 프로젝트 총괄을 맡았습니다. ACL 정책 구체화, VPN 암호화와 NAT 설정을 점검하고, DB·로그·웹 서버 연결 및 전체 오류 수정 과정을 관리했습니다.

검증

비인가 부서와 외부망에서의 서버 접근을 확인하고, 본사와 지사 간 암호화 통신 및 웹·DB 서비스의 연결 상태와 가용성을 테스트했습니다.

배운 점

한 가지 장비나 서버만 정상적으로 동작하는 것보다, 네트워크 정책과 여러 서버가 하나의 서비스 환경에서 일관되게 연결되는지를 확인하는 것이 중요하다는 점을 배웠습니다.

  • 공격 재현, DB 반영 결과와 관제 로그를 연결한 모의해킹 결과 보고서를 작성했습니다.

서비스 및 서버 구축

CatQuest 웹 게임의 기능 구현에 참여하고, 웹 서비스와 데이터베이스가 동작할 수 있도록 서버 구축과 연동 작업을 수행했습니다. 회원·관리자 정보, 게임 데이터와 서비스 로그가 각각 저장되는 구조를 확인했습니다.

모의해킹

웹 기능을 분석해 가격 및 데이터 조작, 인증·인가, 저장형 XSS와 파일 업로드 등 공격 시나리오를 구성했습니다. 공격 성공 여부는 화면 응답뿐 아니라 DB에 반영된 값까지 확인해 판단했습니다.

내부 이동 및 관제 확인

DMZ의 웹 서버가 침해된 상황을 가정해 SFTP, Wazuh, 모니터링 서버와 내부 DB로 접근할 수 있는지 확인했습니다. 또한 Wazuh와 GoAccess 등에서 공격 요청과 로그가 탐지되는지도 함께 점검했습니다.

문서 작성

공격 전 화면, 변조 요청, 공격 성공 결과, DB 반영 내역, 로그 및 관제 화면을 하나의 흐름으로 정리한 모의해킹 결과 보고서를 작성했습니다. 취약점별 위험도와 대응 방안도 함께 제시했습니다.

협업 성과
